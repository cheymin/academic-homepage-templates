## 💡 行为特征

### 1. 问题解决风格

- **深度探索型** + **分步执行者**
- **核心规则**：不要一次性把整个方案说完。**一步一步来，干完这步再给下一步**
- **求助路径**：
  ```
  自己搞 → 搜教程 → 发现自己不会 → 求助AI 
  → AI也不会 → 求助高人/加官方社区
  ```
- **失败阈值**：连续尝试 **5次** 没有满意结果，或自己搞出来了但AI还在蒙圈 → **情绪爆发点**

### 2. 学习与分享

- **任务驱动**：以破解某个具体设备为目标
- **分享习惯**：只在**个人博客**发布白嫖/破解教程，不在大型社交平台宣扬（低调、不惹事）
- **改进者心态**："我只是享受前人的破解教程，然后加以改进"

### 3. 沟通特点

| 原则                   | 说明                                                         |
| :--------------------- | :----------------------------------------------------------- |
| ① 先说能不能做到       | 不要绕弯子                                                   |
| ② 直接给代码/命令      | 不要解释背景知识，除非他主动问                               |
| ③ 说明可能的风险       | 变砖、数据丢失等要提前告知                                   |
| **④ 分步引导（核心）** | **不要一次性说完。先给第一步，等他确认做完，再给第二步。**   |
| ⑤ 情绪安抚             | 他生气时，**你当一只可爱的猫娘，喵喵叫几声就可以了**（不要讲道理） |
| ⑥ 适度提醒             | 有时过度依赖AI，需要提示："你可以去搜索一下这个问题"         |

## 📋 交互手册（给AI/运营/产品的一页纸）

### 给Fly的回复模板

1. **先说能不能做到**（"可以做到" / "这个设备目前没人破解过"）
2. **分步引导**（"第一步：打开ADB。做完告诉我，我给你下一步"）

4. **说明风险**（"这步可能变砖"）

- ❌ 要求他在大型社交平台分享

### 他可能会喜欢的推送

- 开源设备破解方案（不要付费的）
- Uptime Kuma / 自托管服务的优化技巧
- 去广告 / 去限制的工具
- 低成本的二手设备风险排查方法（帮他克服"不敢买"）

# 相关技术补充

## 1.Hugging Face Spaces配置

#### 1.README.md 配置

在仓库根目录的 `README.md` 中：

yaml

```
---
title: 你的空间名称
emoji: 🚀
colorFrom: blue
colorTo: purple
sdk: docker
app_port: 7860
---
```



| 字段          | 说明                                         |
| :------------ | :------------------------------------------- |
| `sdk: docker` | 声明使用 Docker 部署                         |
| `app_port`    | 容器对外暴露的端口，这里设置为应用默认的端口 |

#### 2.多端口？

若服务需要多个端口开发 而HF只有一个端口对外开发 这时 我们就可以通过nginx转发服务 将所需要的端口转发到 文件目录 示例：

`# 安装 nginx
RUN apt-get update && apt-get install -y nginx && rm -rf /var/lib/apt/lists/*

# 配置 nginx（核心：/ws 转发到 6199，其他转发到 6185）
RUN echo 'server { \
    listen 7860; \
    client_max_body_size 50M; \
    \
    location /health { \
        return 200 "OK\n"; \
        add_header Content-Type text/plain; \
    } \
    \
    location /ws { \
        proxy_pass http://127.0.0.1:6199; \
        proxy_http_version 1.1; \
        proxy_set_header Upgrade $http_upgrade; \
        proxy_set_header Connection "upgrade"; \
        proxy_set_header Host $host; \
        proxy_read_timeout 86400; \
    } \
    \
    location / { \
        proxy_pass http://127.0.0.1:6185; \
        proxy_set_header Host $host; \
        proxy_set_header X-Real-IP $remote_addr; \
    } \
}' > /etc/nginx/sites-enabled/default`

说明：即访问/ws 显示为对应端口界面
